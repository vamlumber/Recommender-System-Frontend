REPORT



OBJECTIVE

To build a recommender system for books using user reviews,rating and interaction. The dataset used for the project is scraped from goodreads website by UCSD Prof. Julian Mcauley. Recommender systems over the web are very prominent these days as the ablitiy of the user to browse the whole catalog of items is not possible . These systems provide more accessiablity to the user and also help in business revenue to any organisation.


OVERVIEW

The first phase of the project is to develop the search functionality using the TFIDF model. The dataset has the following schema:

  • "title", "description", "#reivews", "#ratings", "average_rating"
  • "isbn", "country_code", "language_code", "num_pages", "publication_day", "publication_month", "publication_year", "url"
  • "series", "authors", "publisher", "work_id"
  • "popular_shelves": top user-generated shelves for a book, used to define genres by goodreads

This dataset was collected in late 2017 on goodreads.com, where we only scraped users' public shelves, i.e. everyone can see it on web without login. User IDs and review IDs are anonymized. For each book in each user's shelves, we extracted if the book had been read and if it had been rated (and the associated rating score). Thus for each user-book pair, we obtained the following interaction chain: shelved -- read -- rated (and rating score).

In the first phase , we create a document term matrix after lemmentization and stemming . Then we calculate the Term Frequency and Inverse Document Frequency for each of the terms. When the system is hit with a search query , we sanitize (lementization and stopwords) the query ,the for each term, get the tfidf of each term and find the cosine similarity of the query with our model. Thus , we get the ranking of the document for the query .

ENVIRONMENT

The environment used to develop the applicaton. I have used angular to develop the frontend framework to develop the frontend and used flask at backend to provide the the REST API . I have hosted my service on Google Cloud App engine . I have used google storage to save my model dumps with the help of Joblib library.
